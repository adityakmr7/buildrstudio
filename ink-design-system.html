Do & Don't
Principles to keep the system consistent
✓ Do
✓Use semantic tokens (--text-1, --fill), never raw hex
✓Use bold type weight (700–800) for all headings
✓Keep illustrations monochrome — black stroke, no fill colors
✓Use negative space generously — let content breathe
✓Buttons should always be full width on mobile CTAs
✓Use DM Sans across all products in this system
✓Transition color/background with .3s ease for theme switch
✕ Don't
✕Add color accents (blue, green, red) unless for status only
✕Use gradients — they break the monochrome identity
✕Mix multiple font sizes on one card — pick one heading level
✕Use shadows as primary separation — use borders instead
✕Use rounded corners below r-sm (8px) — looks sharp/harsh
✕Place more than 4 items in the bottom navigation
✕Add decorative icons just for visual fill — every element earns its place
